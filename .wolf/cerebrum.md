## Key Learnings — PWA

- **Phase 4 PWA Polish (2026-05-17):** Safe-area insets require `viewport-fit=cover` in viewport meta — without it, `env(safe-area-inset-*)` always resolves to 0. Topbar uses `min-height + padding-top: env(safe-area-inset-top)` instead of `height` to accommodate notch on iPhone standalone. `.view-panel` uses `padding-bottom: calc(32px + env(safe-area-inset-bottom, 0px))`. Mobile fixed sidebar gets `padding-top: env(safe-area-inset-top)` inside the `@media (max-width: 900px)` block.
- **Dark flash prevention pattern:** Add inline `<script>` in `<head>` (before any CSS link) that reads `dandyDarkMode` from localStorage and sets `document.documentElement.style.background = '#0B2B1F'` + `colorScheme = 'dark'`. This sets the html-level bg before body renders. Body's `transition: background .4s` smooths the transition when `body.dark` is applied later by `applyDarkMode()`. No CSS selectors needed.
- **PWA manifest: `background_color` must match `--c-brand-bg` token (`#EDF6F1`):** Was `#EEF2EF` (off). Correct value is `#EDF6F1`. Add `scope: './'`, `lang: 'en'`, `id: './'`, `prefer_related_applications: false`. Icon needs two entries: `purpose: 'any'` and `purpose: 'maskable'` (maskable icon centers content in safe zone — no rounded corners, no transparency).
- **`* { -webkit-tap-highlight-color: transparent; }` is safe for clinical PWAs:** Removes grey iOS tap flash without affecting keyboard focus states or `focus-visible` outlines. Add at top of CSS block.
- **`height: 100dvh`** (dynamic viewport height) prevents layout shift on mobile when address bar appears/disappears. Use `height: 100vh; height: 100dvh;` — browsers that don't support `dvh` fall back to `vh`. `100dvh` = exact visible viewport height.
- **`.page overscroll-behavior-y: contain`** prevents the iOS rubber-band bounce from propagating from the `.page` scroll container to the whole window — keeps scroll within the panel. Pair with `overscroll-behavior: none` on body.

- **Phase 4 script deferral pattern (2026-05-16):** Auth/sync scripts (supabase, authModule, serializer, syncQueue, cloudSync, prefsSync, notesEncryption) are loaded with `defer`. The AI/render scripts (calcAI, clinicalEngine, costEngine, all render panels) remain blocking because the inline body script's `init()` IIFE calls them synchronously. Slot mappings: slot1→'implant', slot2→'bridge', slot3→'crown'.
- **Preservation bias threshold is 3 points.** In `recommend()`, `bestPreserve.slot` wins over `extractOpt.slot` only when `bestPreserve.score >= extractOpt.score - 3`. Scenarios must be calibrated so the gap is ≤3 (bias fires) or >3 (extract wins). This is the key lever for COMPROMISED cases.

## Key Learnings — Graph Density Calibration (2026-05-18)

- **3-option dense SVG collision has three simultaneous causes:** (1) Floating `★ Rec` badge at `recEndY - 6` is an independent text node with no collision awareness — it becomes the 4th text item competing for vertical space. (2) All endpoint labels are placed at `endY + 3.5` independently with no gap enforcement. (3) Fixed H=160 gives plotH=110 → 2.2px/unit; when 3 options finish within a 10-unit Y spread, that's only ~22px for 3 labels + 1 badge.
- **Adaptive height pattern for SVG panels:** Compute the option count BEFORE creating the SVG so the viewBox uses the correct H. `const H = isThreeOption ? 180 : 160`. - **Inline ★ suffix eliminates the floating badge node entirely:** Instead of a separate `<text class="graph-rec-badge">` positioned above the endpoint, append `★` to the recommended option's endpoint value label (`82★` instead of `82`). This removes one text element from the vertical stack in the right margin, eliminating the badge-to-adjacent-label collision class entirely.
- **Greedy downward push-apart for label collision avoidance:** Sort `lbEntries` by screen-y (ascending), then: `for i=1..N: if lbEntries[i].y - lbEntries[i-1].y < MIN_GAP, push lbEntries[i].y = lbEntries[i-1].y + MIN_GAP`. Clamp at `PAD_T + plotH + 2`. MIN_GAP=11 (font-size 9 labels need ~9px, +2px breathing room). Works for N=2 and N=3 with zero additional logic.
- **SVG migration guard handles stale badge-element SVGs:** `svg.querySelector('.graph-rec-badge')` detects SVGs created by the old code. Including it in the invalidation check ensures a clean badge-free SVG is created on the first run after upgrade. After that, no `.graph-rec-badge` elements ever exist, so the check is always false and the SVG is reused normally.
- **Label x position is a constant within a height mode.** `PAD_L + plotW + 5 = 265` is constant for any given H (PAD_L=42, PAD_R=60 don't change). Set `x` at SVG creation time via `LABEL_X = String(PAD_L + plotW + 5)`. The update loop only needs to write `y` and `textContent` — no redundant x setAttribute calls.
- **Crown points must be computed BEFORE the SVG creation block** so that `isThreeOption = !!(ai.crownViable && ai.crown > 0)` is available for the H decision. `crownPts` is also reused in the collision-avoidance section for `cEndY = sy(crownPts[lastIdx])`.

## Do-Not-Repeat

- [2026-05-18] **`genCaseNum()` must use a max-based approach, NOT `pts.length + 1`.** Count-based reissues numbers after deletion: [Case 1, Case 2] → delete Case 1 → [Case 2] → next `pts.length + 1 = 2` → duplicate Case 2. Fix: reduce over all existing patients to find the highest N in any `"Case N"` caseNum, then return `N+1`. Non-`Case N` formats contribute 0 to the max and are silently skipped. This is deletion-safe, import-safe, and deterministic.

- [2026-05-18] **`initPatients()` demo seed must NOT occupy a case number for fresh users.** For fresh installs (no `STORAGE_KEY` migration data), the seed patient is the "Mohamed A." demo from `DEFAULT_STATE`. Assigning it `caseNum: 'Case 1'` offsets all numbering by 1 — the user's first real patient becomes "Case 2". Fix: detect `isMigration` flag; fresh users get `caseNum: null`. Lazy assignment in `saveState()` assigns the next available case number when the patient is first saved. `renderPatientList()` sidebar meta must handle null caseNum: use `${p.caseNum ? escapeHtml(p.caseNum) + ' · ' : ''}${score}` to avoid orphan bullet. Migrated users are unaffected — their seed IS their real first patient and keeps `caseNum: 'Case 1'`.

- [2026-05-18] **`showSkeleton()` destroys the AI card structure without clearing `dataset.built` — guard in `buildAICardStructure` fires on next render, leaving skeleton forever.** Root cause: `showSkeleton('aiCardBody')` replaces innerHTML but leaves `el.dataset.built === '1'`. When `render()` fires in the `switchPatient` 300ms timeout, `buildAICardStructure()` sees `built === '1'` and returns early; `updateAICard()` silently fails because its DOM targets no longer exist. Fix: `delete el.dataset.built` inside `showSkeleton()` after innerHTML replacement. Also: `buildAICardStructure()` must call `body.removeAttribute('aria-busy')` before injecting innerHTML to clear the skeleton's accessibility state.

- [2026-05-15] **`const`/`let` top-level classic-script declarations are NOT `window.*` properties.** In Playwright (and in browsers), `window.X` only resolves for `var` declarations and `function` declarations at script top level. `const ClinicalEngine = ...` makes `ClinicalEngine` available in the global scope but NOT as `window.ClinicalEngine`. Use `typeof X !== 'undefined'` (not `window.X`) when checking for `const`/`let` globals in `page.waitForFunction` or `page.evaluate`. For CI test sentinels, prefer a known function declaration (`window.render`) over `const` globals.
